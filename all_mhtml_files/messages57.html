Things to KEEP IN MIND!

- There are 3 ways to sign the Flutter Android app in Appcircle

1. Auto-generating keystore file by providing alias passwords and name inside the dashboard of Appcircle
2. Uploading already generated keystore file with passwords to Appcircle
- MAKE SURE, your app was not signed already inside the app codebase
- leave it as it was generated with flutter create command
3. If you have already signed the app inside the codebase, you do not have to do anything with singing inside Appcircle
- Workflow automatically signs the app, HOWEVER keep in mind that you have to provide key.properties and keystore file as these files usually are not pushed to repos. You can push these files if you think they are in safe place or you can find more advanced ways inside Appcircle Workflow Editor to get these files
